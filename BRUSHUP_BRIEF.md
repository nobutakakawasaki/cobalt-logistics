# ブラッシュアップ ブリーフ #1（高優先2点 + インタラクション強化）

評価レポート（機能比較表）で高優先とした2点の実装と、参考サイトのようなカーソル
追従インタラクションの追加。DESIGN_BRIEF.md の内容・既存ページ構成はそのまま、
追加・修正のみ行う。

## 1. お問い合わせフォームの実装（高優先）

現状 HOME ページの `#contact` セクションは電話番号・メールのテキスト表示のみで、
送信機能がない。これを実際に動く問い合わせフォームに置き換える。

- カスタム投稿タイプ `inquiry`（問い合わせ）を `functions.php` に登録する。
  - `public` は false、`show_ui` は true にして管理画面の左メニューから確認できるようにする
  - 管理画面の一覧に「氏名・会社名・メールアドレス・お問い合わせ内容の抜粋・送信日時」が
    見えるようカスタムカラムを追加する
- フォーム項目: お名前(必須) / 会社名(任意) / メールアドレス(必須) / お問い合わせ内容(必須、
  テキストエリア) / お問い合わせ種別(セレクト: 資料請求 / 倉庫見学 / その他)
- 送信処理: `admin-post.php` 経由（`admin_post_cobalt_submit_inquiry` /
  `admin_post_nopriv_cobalt_submit_inquiry`）。
  - WPノンス（`wp_nonce_field` / `check_admin_referer`）で保護する
  - サーバー側で必須項目チェック・`sanitize_text_field` / `sanitize_email` /
    `sanitize_textarea_field` でサニタイズする
  - 検証OKなら `wp_insert_post` で `inquiry` 投稿として保存する
  - 保存に加えて `wp_mail()` の送信も試みてよいが、このDocker環境にはメール送信手段が
    ないため失敗しても構わない（≒ベストエフォート。失敗してもユーザー側には「送信できました」
    と表示してよい。実運用では管理画面から内容を確認できるので実害はない、という前提）
  - 処理後は HOME ページの `#contact` にクエリパラメータ付きでリダイレクトし、
    成功時は「お問い合わせありがとうございます」、入力不備時はエラー内容を表示する
  - JSでのクライアント側バリデーション（必須項目・メール形式）も入れて、送信前に
    気づけるようにする
- 既存の電話番号・メール表示はフォームの下に残してよい（フォームを使わない人向けの導線として）

## 2. 地図をダミー枠から実装に差し替え（高優先）

倉庫概要ページ（`page-warehouse.php`）の `.map-placeholder` を、Google マップの
iframe埋め込みに差し替える。

- **住所ピンポイントでの検索は行わないこと。** 番地まで指定すると実在の建物が
  ピンで表示されてしまい、架空の会社と無関係な実在施設を指し示す形になるため。
  代わりに地域レベル（例:「横浜市鶴見区大黒町」程度、番地なし）でGoogleマップの
  `output=embed` を使い、地域の地図が表示される程度に留める。
- 地図の直下または上に一言キャプションを添える:
  「所在地イメージ（デモ用のため実際の建物とは異なります）」
- iframeには `loading="lazy"`、`title` 属性、適切な `width="100%"` のレスポンシブ
  ラッパー（aspect-ratio指定でよい）をつける。

## 3. カーソル追従インタラクションの追加（参考サイト風の"動き"）

参考にしたサイト（stockcrew.co.jp）はマウスカーソルの動きに合わせて要素が動く
インタラクションがあり、「かっこいい」との評価があった。以下を vanilla JS + CSS のみで
（外部ライブラリ不要）実装する。既存の `js/main.js` に追記してよい。

- **HOMEのファーストビュー（hero セクション）**: 背景にコバルトブルー系の柔らかい
  グラデーション/ぼかし円形（blob）を1〜2個配置し、マウス移動に応じて
  `transform: translate()` で緩やかに追従させる（即座に追従せず、`requestAnimationFrame`で
  イージングをかけて"遅れてついてくる"動きにする）。派手すぎない、上品な動き量に抑える。
- **サービスカード（HOME・サービス概要ページ）**: ホバー時にカーソル位置に応じて
  軽くtilt（傾き）する、またはカード内のアイコンがカーソル方向に少し寄る、程度の
  控えめなマイクロインタラクション。
- **必須の配慮**:
  - `@media (prefers-reduced-motion: reduce)` では上記のマウス追従アニメーションを
    無効化し、静的な表示にフォールバックする
  - タッチデバイス判定（`matchMedia('(pointer: coarse)')` 等）でカーソル追従は
    スキップする（タッチ端末ではマウスムーブイベントが発火しないため実質的に無害だが、
    念のためリスナー登録自体をスキップして良い）
  - `mousemove` リスナーは `requestAnimationFrame` でスロットルし、パフォーマンスに
    配慮する
