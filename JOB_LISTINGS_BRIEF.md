# 求人詳細ページ ブリーフ #6

採用情報ページ(`page-recruit.php`)の求人カードは現状クリックできない静的なリストです。
これを実際の求人サイト(Amazonの物流拠点求人などが参考モデル)のように、各求人をクリックすると
詳細（仕事内容・必須スキル・歓迎スキル・求める人物像・給与等）が見られる形に拡張する。

## 技術方針: `job` カスタム投稿タイプを新設する

`inquiry` CPT（`functions.php` に既存）と同じ要領で、新たに `job` という投稿タイプを追加する。
`public => true`, `has_archive => false`（一覧は既存の `/recruit/` ページに残すため、WPの
自動アーカイブは不要）, `rewrite => array('slug' => 'job')` で `/job/<slug>/` 形式のURLにする。

### 保存するデータ

各求人は通常のWP投稿として保存し、以下をpost meta（`register_post_meta`は不要、
`update_post_meta`で保存し`get_post_meta`で読めればよい、ACF等のプラグインは使わない）で持たせる:

- `job_type`（雇用形態、例: "正社員"）
- `job_location`（勤務地）
- `job_salary`（給与）
- `job_summary`（仕事内容、複数行の説明文）
- `job_required`（必須スキル・経験、改行区切りのテキスト。テンプレート側で1行ずつ`<li>`に変換する）
- `job_preferred`（歓迎スキル・経験、同上）
- `job_profile`（求める人物像、同上）

## 実装内容

### 1. `functions.php`
`cobalt_logistics_register_inquiry_cpt` と同じ場所に `job` CPTを登録する関数を追加し、
`init` フックに登録する。管理画面用のUIも出す（`show_ui => true`、`inquiry` と違い
`job` は編集・閲覧の対象になるコンテンツなので、`show_in_menu => true` でよい）。

### 2. `single-job.php`（新規テンプレート）
既存の下層ページ（`page-warehouse.php` 等）と同じ構成パターンに揃える:
- `.page-hero` 相当のヘッダー（求人タイトル、雇用形態・勤務地・給与をまとめたメタ情報）
- 「仕事内容」セクション（`job_summary`）
- 「必須スキル・経験」セクション（`job_required` を箇条書きで）
- 「歓迎スキル・経験」セクション（`job_preferred` を箇条書きで）
- 「求める人物像」セクション（`job_profile` を箇条書きで）
- 応募CTA（`cobalt_logistics_page_url('home') . '#contact'` へのボタン、
  文言は「この求人に応募する」等）
- 採用情報一覧に戻るリンク（`cobalt_logistics_page_url('recruit')`）

新しいCSSクラスは既存トークン（`--paper` `--ink` `--line` `--cobalt` 等）に揃え、
箇条書きスタイルは他ページの `.spec-list` 等を参考にしてよいが、無理に流用せず
読みやすさを優先してよい。

### 3. `page-recruit.php` の「募集職種」セクション
現状ハードコードされた5枚の `.job-card` を、`WP_Query`（`post_type => 'job'`,
`orderby => 'menu_order'` または `post__in` で上記5件の順序を維持）でループする形に
置き換える。各カードを `<a>` タグで囲み、対応する求人の `get_permalink()` へリンクさせる。
既存の `.job-card` の見た目（タイトル・勤務地・雇用形態バッジ）は維持したまま、
カード全体をクリック可能にする（`:hover` の視覚フィードバックも追加するとよい）。

### 4. データ投入
5件の求人をWP-CLIで作成する（`wp post create --post_type=job --post_title=... --post_status=publish`
の後 `wp post meta update` でメタを設定、またはPHPスクリプトで一括投入してもよい）。
既存の `page-recruit.php` にハードコードされていた3件（倉庫内オペレーションスタッフ、
物流管理職、配送コーディネーター）と内容が重複しないよう、投入後は元のハードコード分を
削除すること（上記5件データがそのまま新しい正のソースになる）。

## 制約

- 既存の機能（問い合わせフォーム・地図・カーソル追従・ルートライン・シミュレーター・
  スクロールリビール・導入事例・サービス画像・追従CTAバー・ヒーロースライドショー・
  社員の声・選考の流れ）を壊さないこと。
- vanilla PHP/CSSのみ、ACFやカスタムプラグインは使わない。
- レスポンシブ対応必須。
- アクセシビリティ: 見出し階層を適切に保つこと（`single-job.php` の `<h1>` は求人タイトルのみ）。
