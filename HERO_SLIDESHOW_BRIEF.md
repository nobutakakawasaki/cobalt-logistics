# ヒーロー写真スライドショー ブリーフ #5

参考サイト(stockcrew.co.jp)のソース調査で見つけた「hero-parapop」（画像がポップイン→
ゆっくりドリフト→吸い込まれるように退場、を繰り返す）演出を、vanilla JS/CSSで
自分たちの実装として再現する。GSAPは使わない（このテーマの既存方針どおり）。

## 画像素材(生成済み、`wp-content/themes/cobalt-logistics/assets/images/` に配置済み)

- `hero-slide-1.jpg` — 倉庫内観、夜、青いLEDラインが棚に沿って光る、ドラマチックな暗め構図
- `hero-slide-2.jpg` — 倉庫外観の空撮、夕景、荷捌きバースに整列したトラック群
- `hero-slide-3.jpg` — コンベアで搬送される段ボールのクローズアップ、モーションブラーで
  スピード感を演出

いずれもAI生成画像（架空の情景、実在ブランド・識別可能な人物の顔なしを確認済み）。

## 配置場所

`page-home.php` の `.hero` セクション。現在は `linear-gradient` の単色背景の上に
`.hero-blob`(カーソル追従)と `.hero-route`(ルートラインSVG)が乗っている構成。
**この2つは削除・変更しない。** 写真スライドショーはその「下」の新しい背景レイヤーとして
追加する（重なり順: 写真スライドショー → コバルトブルーのグラデーションオーバーレイ →
既存のhero-blob → 既存のルートラインSVG → `.hero__inner`(テキスト/マニフェストリスト)）。

## 挙動

1. 3枚の画像を絶対配置で重ね、`.is-active` クラスが付いた1枚だけを表示する
   （`object-fit: cover` で `.hero` いっぱいに敷き詰める）。
2. サイクル（JSの `setInterval` またはタイムアウトの連鎖で駆動、GSAP不要）:
   - **ポップイン**: 現在の画像が `opacity: 0` かつ僅かに縮小した状態(`scale: 0.94`程度)
     から、`opacity: 1` / `scale: 1` へ、約800〜900msでトランジション
   - **ホールド+ドリフト**: 表示中の約4〜5秒間、`scale` がごくゆっくり `1` → `1.04` 程度まで
     クリープするKen Burns風の演出（CSSの `transition` の duration を長く取るだけでよい。
     JSでの毎フレーム計算は不要）
   - **退場**: `opacity: 1 → 0` で約600msかけてフェードアウト。次の画像のポップインは
     退場が終わってから始める（クロスフェードの重なりは不要、順番に切り替わればよい）
   - 3枚をこの順で無限ループ
3. **パフォーマンス配慮**: `document.visibilityState` を見て、タブが非アクティブな間は
   タイマーを止める（余計なバックグラウンド処理をしないため）。
4. **オーバーレイ**: 写真の上に、既存の `--cobalt` / `--cobalt-deep` を使った半透明の
   グラデーション（例: `linear-gradient(135deg, rgba(12,58,140,.92), rgba(30,95,204,.6))`
   程度の値は実装者調整でよいが、方向は既存 `.hero` の135degグラデーションに揃える）を
   重ね、テキストのコントラストとブランドカラーの統一感を保つ。既存の `.hero__title`
   `.hero__lead` `.hero-manifest` の可読性が写真背景に負けないよう必ず確認すること。
5. **アクセシビリティ**: 3枚とも装飾目的の背景画像（固有の情報を持たない）なので
   `alt=""` を明示的に設定し、スライドショーのコンテナに `aria-hidden="true"` を付ける
   （スクリーンリーダーが読み上げる情報が何もないことを明示）。
6. **reduced-motion**: `prefers-reduced-motion: reduce` では1枚目の画像を静止表示のまま
   固定し、切り替え・ドリフト・ポップインアニメーションを一切行わない
   （JS側でタイマー自体を起動しない。CSS側でも `transition`/`animation` を `none` に
   するバックストップを入れる。既存の `.reveal` 等のパターンに準拠）。

## 制約

- GSAP等の外部ライブラリは使わない。vanilla JS/CSSのみ。
- 既存の `.hero-blob`(カーソル追従)・`.hero-route`(ルートラインSVG)・
  `.hero__content`・`.hero-manifest`・問い合わせフォーム・地図・シミュレーター・
  導入事例・サービス画像・追従CTAバーを壊さないこと。
- レスポンシブ対応必須（SPでも同じ挙動でよいが、`object-fit: cover` で崩れないこと）。

- reduced-motionでタイマーが起動しないことをコードレベルで確認

- `.hero__title` 等のテキストが写真背景の上でも十分なコントラストで読めること
  （オーバーレイの不透明度を目視レベルで確認できないため、使用した色のアルファ値と
  コントラスト計算の根拠を報告に含めること）
